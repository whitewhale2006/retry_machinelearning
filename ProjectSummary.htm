Machine Learning Class Project

1.       Introduction of the data

Human Activity Recognition - HAR - has emerged as a key research area in the last years and is gaining increasing attention by the pervasive computing research community (see picture below, that illustrates the increasing number of publications in HAR with wearable accelerometers), especially for the development of context-aware systems. There are many potential applications for HAR, like: elderly monitoring, life log systems for monitoring energy expenditure and for supporting weight-loss programs, and digital assistants for weight lifting exercises. A dataset with 5 classes (sitting-down, standing-up, standing, walking, and sitting) is proposed and collected on 8 hours of activities of 4 healthy subjects.

Read more: http://groupware.les.inf.puc-rio.br/har#ixzz4DBS2I1kv

One baseline performance index is also established.

2.       Project Goal

The goal of this project is to develop the classification rules using the training data. The classification rules are applied to segment the new observation into the appropriate “classe” categories.

For the data validation purpose, the original training data is split to be two parts. 2/3 of the train data is used to train the model, and the remaining 1/3 train data is used to validate the model. If the methodology is appropriate, the validation results should have low error rate for the testing data. Once the model is proved to be efficient, it is applied to segment the test data into the corresponding “classe” category.  Many variables in the train data have missing values. To clean the data and select the most relevant variable, some variables with high missing rate and the variable having different values in train and test datasets are dropped directly instead of imputing the missing values. Eventually, 56 variables and 13,083 observations left in the new training data set; 56 variables and 6539 observations left in the new validating data set.
